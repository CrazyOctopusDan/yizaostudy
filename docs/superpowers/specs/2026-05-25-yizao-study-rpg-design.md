# 一级造价师 RPG 学习计划系统设计

## 背景

目标用户准备参加 2026 年一级造价工程师考试，所在地为浙江省嘉兴市，专业方向为土木建筑工程。用户已取得二级造价师，并从事造价行业，因此第一版不按零基础教学工具设计，而是按备考推进、习惯养成、真题训练和错题复盘设计。

2026 年一级造价工程师考试时间按全国考试计划倒排为 2026-10-17 至 2026-10-18。浙江嘉兴相关报名、资格审核、准考证打印等提醒后续以浙江人事考试网当年考务通知为准。

正式工程目录为 `/Users/zhangyi/zhangyi/homework/yizaostudy`。当前目录仅用于设计讨论和规格文档沉淀。

## 产品目标

第一版做一个 Windows 本地可运行的学习推进网页应用。用户拿到压缩包后，在 Windows 电脑上解压，双击启动脚本，即可打开本地网页进行每日学习。

核心目标是每天打开后能清楚回答：

- 今天必须完成什么。
- 当前距离考试还有多久。
- 四科中已勾选科目的进度和风险如何。
- 完成或漏做任务后，下一步该怎么调整。

第一版优先保证本地快速落地，不做公网账号、多用户协作、移动端优先、AI 推荐、粉笔账号直连、付费平台破解或批量搬运题库内容。

## 交付形态

第一版交付为一个可压缩的 Windows 本地项目包，包含：

- 应用代码。
- 锁定依赖文件。
- Windows 启动脚本 `启动学习计划.bat`。
- 简短使用说明。
- 本地数据目录。

用户双击 `启动学习计划.bat` 后，脚本启动本地网页服务并打开默认浏览器。如果 Windows 电脑没有安装 Node.js 或所需运行时，脚本应给出清晰提示。后续版本可以升级为内置运行时或桌面 App，以进一步接近真正的解压即用。

数据保存在解压目录下的本地数据库文件中，便于备份、迁移和打包。公网访问不是第一版目标，但技术结构不应阻碍后续增加登录、HTTPS、备份和反向代理。

## 页面结构

第一版包含 5 个主页面。

### 今日学习

默认首页，采用 `Cozy Quest + Adventure Map` 的 RPG 风格。页面展示：

- 今日必做、建议、可放弃任务。
- 考试倒计时。
- 用户角色等级、经验值和连续学习天数。
- 已勾选科目的进度、风险和下一步推荐。
- 冒险地图形式的阶段节点。

首页要温暖、卡通、低压力，但不能弱化考试风险提示。

### 学习计划

展示从当前日期到考试的阶段计划，按习惯期、系统学习期、强化期、冲刺期组织。用户可以：

- 勾选本轮备考科目。
- 查看周计划。
- 调整每周学习强度。
- 触发未来任务重排。

四科先全部列出：建设工程造价管理、建设工程计价、建设工程技术与计量（土木建筑工程）、建设工程造价案例分析（土木建筑工程）。系统只对已勾选科目生成任务。

### 题目练习

支持三个入口：

- 今日任务推题。
- 按科目和年份刷真题或套卷。
- 错题本复盘。

题目可手动导入，字段包括题干、选项、答案、解析、科目、知识点、年份、来源链接和错题状态。第一版不批量采集整站内容。

### 资料库

管理官方教材章节、网课链接、电子资料链接、本地 PDF 或文件路径、个人备注。学习任务可以绑定资料，用户在今日任务中点击即可打开对应链接或文件位置。

### 复盘统计

展示学习连续天数、任务完成率、科目进度、错题分布、薄弱知识点、近 7 天和 30 天学习趋势、外部考试成绩记录。

## 游戏化设计

用户有一个角色，等级范围为 `Lv.1` 到 `Lv.100`。学习行为和考试记录可以获得经验值：

- 完成必做任务获得较多经验。
- 完成建议任务获得中等经验。
- 完成可放弃任务获得少量经验。
- 录入外部真题或模拟考试分数后，按科目、分数和难度增加经验。
- 错题复盘和高质量复习可以获得额外经验。
- 单纯打开资料不给或只给极少经验，避免空转升级。

每个科目也有独立的熟练进度或等级，用冒险地图、关卡节点或进度条表达。角色等级用于激励，不能替代真实备考进度。首页仍必须显示考试风险，避免用户通过低价值任务升级却没有真正提升应试能力。

## 计划规则

### 四阶段倒排

系统从当前日期倒排到 2026-10-17：

- 习惯期：低压力建立每日学习节奏，任务少但强调连续。
- 系统学习期：按已勾选科目章节推进。
- 强化期：真题、错题和案例专题训练加重。
- 冲刺期：套卷、案例采分点、错题回炉和考前记忆清单。

### 任务分级

每日任务分为：

- 必做：关键学习或复盘，不完成会进入补偿队列。
- 建议：有余力完成，未完成可顺延一次。
- 可放弃：资料拓展或额外刷题，未完成不追补。

### 动态调整

如果某天未完成任务：

- 必做任务进入补偿队列，未来 3 至 7 天自动分摊。
- 建议任务最多顺延一次，再降级或丢弃。
- 可放弃任务直接跳过。
- 每周生成风险提示：正常、轻微滞后、明显滞后、需缩科目目标。

如果长期未学习，系统生成恢复计划，先用 2 至 3 天低负荷任务恢复节奏，再回到主计划。

### 科目选择

系统初始列出四科，用户使用时勾选本轮备考科目。计划、任务、风险和经验加成只围绕已勾选科目计算。未勾选科目可以保留资料和题目，但不主动生成每日任务。

## 题目与资料来源

第一版把来源分为三类。

### 学习资料

官方教材章节是主线。每个章节可以绑定网课链接、电子资料链接、本地文件路径和个人备注。

### 公开题源索引

系统可以内置或手动维护公开题源链接，例如 233 网校、爱真题等公开页面。记录来源名称、URL、科目、年份、题型和是否土建相关。

系统保存来源索引和用户手动导入的内容，不做未经授权的批量抓取、复制、转载或传播。粉笔不作为第一版直连目标。

### 题目导入与错题

支持手动导入单题或套题。题目缺少答案或解析时可以保存为草稿。错题进入复盘队列，并影响后续任务和科目风险。

## 技术结构

第一版采用轻量本地 Web 应用：

- 前端：单页应用，电脑浏览器优先，RPG 卡通风格。
- 后端：本地服务，提供数据 API、计划生成、任务重排、题目管理和资料管理。
- 数据：本地 SQLite 或等价轻量数据库，保存科目选择、任务、完成记录、经验值、题目、错题、资料链接和外部考试成绩。
- 启动：Windows 双击批处理脚本启动本地服务并打开浏览器。

不应把核心数据只放在浏览器 LocalStorage 中，因为后续需要备份、迁移、压缩包交付和公网化扩展。

## 核心数据流

1. 初次使用时配置考试信息、所在地、土建方向，并勾选备考科目。
2. 系统按考试日期、阶段、科目和任务模板生成学习计划。
3. 首页读取今日任务、倒计时、角色等级、经验值、科目地图和风险提示。
4. 用户完成任务后，系统写入完成记录，增加经验值，更新科目进度。
5. 用户录入外部真题或模拟考试成绩后，系统记录成绩，增加经验值，并更新科目风险。
6. 每天或每周根据未完成任务和正确率重排后续任务。
7. 错题进入复盘队列，未来任务自动插入复盘事项。

## 错误与边界处理

- 未完成任务按必做补偿、建议顺延一次、可放弃跳过处理。
- 未勾选科目不生成任务。
- 外部考试成绩只作为经验和风险参考，不直接标记章节已掌握。
- 题目导入缺答案或解析时允许保存草稿。
- 公开题源失效时保留历史记录并提示链接不可用。
- 本地运行失败时显示缺少运行时、端口占用或依赖异常的清晰提示。
- 公网访问未开启前，不提供账号系统。

## 验收标准

第一版完成时应满足：

- 能在正式工程目录 `/Users/zhangyi/zhangyi/homework/yizaostudy` 中运行和打包。
- 能生成可给 Windows 用户解压的项目压缩包。
- Windows 用户可通过 `启动学习计划.bat` 启动本地网页。
- 能完成初始配置：考试日期、浙江嘉兴、土建方向、勾选科目。
- 首页能显示今日任务、倒计时、角色等级、经验值、科目地图和风险提示。
- 完成任务后，经验值、任务状态和科目进度能正确变化。
- 漏做任务后，必做任务能进入补偿队列。
- 能录入外部真题或模拟考试分数，并按规则增加经验值。
- 能新增资料链接和题目。
- 错题能进入复盘队列。
- 本地数据能随解压目录保留，便于备份和迁移。
